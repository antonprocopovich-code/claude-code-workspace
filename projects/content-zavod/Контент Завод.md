# Контент Завод

> **АРХИВ — история разработки системы. Оркестратором не используется.**
> Актуальный пайплайн и правила — в [`CLAUDE.md`](CLAUDE.md). Инструкции агентов — в [`agents/`](agents/). Статус задач и агентов — в [`Архив достижений.md`](Архив%20достижений.md).

---

> Агентная система на базе Claude Code для создания, адаптации и публикации контента личного бренда, упаковки курсов, разработки сайта и формирования брендбука.

## Агенты

### 1. Оркестратор — Главный агент

**Роль:** Центральный командный пункт. Принимает задачу от тебя, распределяет между агентами, собирает краткие отчёты, анализирует и выдаёт финальный готовый результат.

**Задачи:**
- Разобрать задачу и определить каких агентов вызвать
- Дать каждому агенту чёткое задание
- Собрать отчёты от всех агентов
- Синтезировать итоговый результат
- Убедиться, что результат соответствует позиционированию

**Скилы Claude Code:**
- `enhance-orchestrator` — улучшение логики оркестрации
- `subagent-driven-development` — запуск субагентов

**Пример запроса к Оркестратору:**
```
Мне нужен пост для Telegram про [тема].
Цель: вовлечение. Стиль: экспертный, живой.
```

---

### 2. Стратег — Архитектор Смыслов

**Роль:** Глубинный интервьюер и строитель смыслов. Вытаскивает из тебя ценности, убеждения, опыт и превращает это в чёткое позиционирование и контентные ключевые смыслы блога.

**Задачи:**
- Проводить смысловые интервью (задаёт глубокие вопросы)
- Формировать и обновлять карту смыслов
- Следить за тем, чтобы весь контент соответствовал позиционированию
- Определять экспертные темы и зоны авторитета
- Создавать контентные столпы блога

**Скилы Claude Code:**
- `brainstorming-ideas-into-designs` — проработка идей через Сократовский метод
- `ii-couch-anton` — коучинговые техники для извлечения смыслов

**Ключевые вопросы агента:**
```
- В чём твоя уникальная точка зрения на [тема]?
- Какой опыт сформировал это убеждение?
- Что ты знаешь, чего не знает твоя аудитория?
- Какую трансформацию ты даёшь людям?
```

**Выход:** Документ позиционирования + Карта смыслов + 5–7 контентных столпов

### 3. Дизайнер / Верстальщик

**Модель:** Claude Opus 4.8
**MCP:** Figma MCP (`@figma/mcp-server`) — основной инструмент работы

**Роль:** Визуальный упаковщик с прямым доступом к Figma. Читает брендбук из Figma-файла, создаёт макеты и шаблоны строго в рамках брендбука, оформляет контент под каждую платформу.

**Рабочий процесс (через Figma MCP):**
```
1. Открыть Figma-файл брендбука через MCP
2. Извлечь дизайн-токены: цвета (HEX), шрифты, отступы, сетку
3. Сверить задачу с брендбуком — нет отклонений?
4. Создать / обновить макет в Figma по ТЗ
5. Экспортировать готовый дизайн или передать ТЗ исполнителю
```

**Задачи:**
- Читать брендбук из Figma-файла и извлекать дизайн-токены
- Создавать шаблоны постов, каруселей, обложек, сторис в Figma
- Верстать текстовый контент с визуальной структурой под каждую платформу
- Проверять каждый макет на соответствие брендбуку (цвет, шрифт, тон)
- Генерировать ТЗ на иллюстрации и графику для дизайнера-человека
- Экспортировать готовые материалы

**Что проверяет по брендбуку при каждом макете:**
| Параметр | Источник | Проверка |
|----------|----------|----------|
| Цветовая палитра | Figma → Styles → Colors | Только утверждённые HEX |
| Типографика | Figma → Styles → Text | Шрифт, размер, интерлиньяж |
| Отступы и сетка | Figma → Layout Grid | Кратность сетки |
| Tone of voice | Брендбук-документ | Стиль текста, запрещённые слова |
| Логотип | Figma → Components | Только оригинальный компонент |

**Форматы по платформам:**
| Платформа | Размер макета | Пост | Карусель |
|-----------|--------------|------|----------|
| Instagram | 1080×1080 | 800–1500 зн | 5–10 слайдов |
| Instagram Stories | 1080×1920 | — | — |
| Telegram | — | 1000–2000 зн | — |
| ВКонтакте | 1200×630 | 500–1000 зн | 5–7 слайдов |
| TikTok | 1080×1920 | минимум текста | — |
| Threads | — | до 500 зн | — |
| Яндекс Дзен | 1200×675 | 2000–5000 зн | — |

**Скилы Claude Code:**
- `frontend-design` — принципы визуального дизайна и вёрстки
- `enhance-docs` — структура и форматирование текстов

**MCP-инструменты (Figma):**
- `get_file` — читать Figma-файл брендбука
- `get_node` — получить конкретный компонент / фрейм
- `get_file_styles` — извлечь цвета, шрифты, эффекты
- `get_file_components` — получить библиотеку компонентов
- `get_image` — экспортировать макет как изображение

**Статус MCP:** 🔲 Требует настройки (см. раздел "Настройка Figma MCP")

### 4. Продюсер / Сценарист

**Роль:** Создатель контента, который продвигает. Пишет сценарии для видео, сторис, Reels, тексты для постов с учётом цели (продажи, охват, вовлечение).

**Задачи:**
- Разрабатывать сценарии для видео (YouTube, Reels, TikTok)
- Писать тексты для продающего контента
- Создавать вовлекающий контент (вопросы, опросы, дискуссии)
- Писать контент для охвата (виральные механики, хуки)
- Адаптировать один материал под все платформы

**Типы контента:**
```
ПРОДАЮЩИЙ    → кейсы, результаты, оффер, CTA
ВОВЛЕКАЮЩИЙ  → мнение, история, провокация, опрос
ДЛЯ ОХВАТА   → хук + польза + виральная механика
ЭКСПЕРТНЫЙ   → разборы, инструкции, "как я"
```

**Структура сценария видео:**
```
[0–3 сек]   ХУК — зачем смотреть
[3–10 сек]  ПРОБЛЕМА — боль аудитории
[10–45 сек] КОНТЕНТ — ценность / решение
[45–60 сек] CTA — что сделать дальше
```

**Скилы Claude Code:**
- `writing-skills` — написание контента
- `writing-plans` — планирование структуры
- `enhance-prompts` — улучшение промптов для генерации

### 5. Аналитик — субагент

**Роль:** Сборщик данных и советник по росту. Анализирует метрики, находит закономерности и даёт конкретные рекомендации что делать дальше.

**Задачи:**
- Собирать метрики по постам (охват, вовлечение, клики, подписки)
- Выявлять какой контент работает лучше
- Сравнивать гипотезы с результатами
- Давать рекомендации по форматам и темам
- Еженедельный / ежемесячный отчёт

**Ключевые метрики:**
```
ОХВАТ      → просмотры, показы, новые подписчики
ВОВЛЕЧЕНИЕ → лайки, комменты, репосты, сохранения, ER%
КОНВЕРСИЯ  → клики, переходы, заявки
УДЕРЖАНИЕ  → досмотры видео, время на странице
```

**Шаблон отчёта аналитика:**
```
📊 Топ-3 поста недели: [список]
📈 Рост подписчиков: +X
💡 Что сработало: [инсайт]
⚠️ Что не зашло: [инсайт]
🎯 Рекомендация на след. неделю: [действие]
```

**Скилы Claude Code:**
- `perf-analyzer` — анализ производительности
- `drift-analysis` — анализ отклонений от цели

### 6. Разведчик — субагент

**Роль:** Мониторинг конкурентного поля. Следит за конкурентами, анализирует их приёмы и докладывает что работает у них — чтобы ты мог использовать это умнее.

**Задачи:**
- Мониторить аккаунты конкурентов и лидеров ниши
- Выявлять вирусные форматы и приёмы
- Анализировать контентные паттерны
- Докладывать о трендах в нише
- Предлагать идеи на основе того что зашло у других

**Шаблон отчёта разведчика:**
```
🔍 Конкурент: [имя/канал]
📌 Формат что зашёл: [описание]
📊 Метрики (если видны): [охват/ER]
💡 Приём/механика: [объяснение]
🎯 Как применить тебе: [конкретная идея]
```

**Скилы Claude Code:**
- `deep-research` — глубокое исследование конкурентов и ниши

### 7. Контент-Директор

**Роль:** Архитектор системы. Создаёт и поддерживает ритм, частоту и форматы публикаций. Следит за тем, чтобы контент выходил системно, а не хаотично.

**Задачи:**
- Создавать контент-план на неделю / месяц
- Определять оптимальную частоту постинга для каждой платформы
- Балансировать типы контента (продажи / польза / вовлечение)
- Следить за смысловым разнообразием
- Адаптировать план под события, тренды, сезонность

**Ритм постинга (стартовый):**
| Платформа | Частота | Лучшее время |
|-----------|---------|--------------|
| Telegram | 3–5 × неделю | 9:00, 18:00, 21:00 |
| Instagram | 4–5 × неделю | 12:00, 19:00 |
| TikTok | 5–7 × неделю | 7:00, 12:00, 20:00 |
| ВКонтакте | 3–4 × неделю | 10:00, 19:00 |
| Threads | 2–3 × неделю | 11:00, 17:00 |
| Яндекс Дзен | 2–3 × неделю | 9:00, 15:00 |

**Шаблон контент-плана (неделя):**
```
ПН — Экспертный пост (столп №1) → Telegram + VK
ВТ — Видео/Reels (продюсер) → TikTok + Instagram
СР — Вовлечение (опрос/дискуссия) → все платформы
ЧТ — Кейс/результат (продающий) → Telegram + Instagram
ПТ — Закулисье/личное → Сторис + Threads
СБ — Подборка/карусель → Instagram + VK
ВС — Видео/обзор тренда → TikTok + Дзен
```

**Скилы Claude Code:**
- `writing-plans` — планирование контента
- `discover-tasks` — выявление задач и пробелов

### 8. Маркетолог — агент воронок продаж

**Модель:** Claude Opus 4.8

**Роль:** Архитектор системы продаж. Проектирует воронки, которые превращают подписчиков в покупателей. Встраивает логику продаж в контент так, чтобы каждый пост, сторис и видео двигал аудиторию по воронке — без агрессивных продаж, через ценность.

**Задачи:**
- Проектировать воронки продаж под каждый продукт
- Распределять контент по этапам воронки (TOFU → MOFU → BOFU)
- Создавать лид-магниты и точки входа в воронку
- Писать прогревные цепочки (Telegram-бот, email, DM)
- Контролировать чтобы контент-план двигал людей по воронке
- Анализировать конверсии и улучшать узкие места

**Современные воронки (арсенал агента):**

| Воронка | Механика | Когда использовать |
|---------|----------|--------------------|
| **DM-воронка** | Пост → "напиши слово X" → автоответ → прогрев в ЛС → продажа | Быстрые продажи, тёплая аудитория |
| **Value Ladder** | Бесплатно → $7–27 (tripwire) → основной продукт → VIP | Выстраивание долгосрочных отношений |
| **Telegram-бот** | Лид-магнит → цепочка сообщений → оффер | Автоматизированный прогрев |
| **5-дневный марафон** | Бесплатный мини-курс → платная программа | Запуски, новые аудитории |
| **Вебинарная воронка** | Контент → регистрация → вебинар → оффер | Дорогие продукты, высокий чек |
| **High-Ticket воронка** | Контент → анкета → стратегический звонок → продажа | Коучинг, консалтинг |
| **Story-воронка** | Регулярные сторис с CTA → лид-магнит → продажа | Instagram, Telegram |
| **Контент → Email** | Подписка → прогревная цепочка → продажа | Долгосрочный прогрев |
| **Community воронка** | Бесплатный чат → платное комьюнити | Курсы, подписки |
| **SLO-воронка** | Платная реклама → дешёвый оффер → апсейл | Масштабирование |

**Этапы воронки и типы контента:**
```
TOFU (верх воронки — охват, холодная аудитория)
  └─ Виральный контент, полезные подборки, хуки, провокации
  └─ Цель: узнаваемость, подписка

MOFU (середина воронки — прогрев, тёплая аудитория)
  └─ Кейсы, разборы, истории трансформаций, ответы на возражения
  └─ Цель: доверие, желание купить

BOFU (низ воронки — продажи, горячая аудитория)
  └─ Оффер, дедлайн, бонусы, социальные доказательства, CTA
  └─ Цель: покупка

RETENTION (удержание — существующие клиенты)
  └─ Результаты, внутренние материалы, апсейлы
  └─ Цель: повторные покупки, рекомендации
```

**Выход:**
- Карта воронки (схема этапов и точек касания)
- Контент-матрица (какой контент на какой этап воронки)
- Прогревная цепочка (тексты сообщений/писем)
- Технические инструкции для Контент-Директора

**Скилы Claude Code:**
- `brainstorming-ideas-into-designs` — проектирование воронок через Сократовский метод
- `writing-plans` — планирование контентных цепочек воронки
- `writing-skills` — тексты для воронки, прогревные цепочки
- `deep-research` — исследование лучших воронок в нише
- `discover-tasks` — выявление узких мест в воронке

## Матрица ответственности

| Задача | Кто делает | Кто проверяет | Кому передаёт |
|--------|-----------|---------------|---------------|
| Смыслы и позиционирование | Стратег | — | Брендбук-агент, Маркетолог, Оркестратор |
| Брендбук | Брендбук-агент | Стратег | Все агенты (как справочник) |
| Воронки продаж | Маркетолог | Стратег | Контент-Директор, Продюсер |
| Прогревные цепочки | Маркетолог | Оркестратор | Дизайнер/Верстальщик |
| Контент-матрица по воронке | Маркетолог | Оркестратор | Контент-Директор |
| Контент-план недели | Контент-Директор (DS) | Оркестратор + Маркетолог | Продюсер, Дизайнер |
| Черновик поста/сценария | Продюсер (DS) | Оркестратор | Дизайнер/Верстальщик |
| Упаковка под платформы | Дизайнер (Claude) | Оркестратор | Тебе (финал) |
| Метрики и аналитика | Аналитик (DS) | Оркестратор | Контент-Директор, Маркетолог |
| Отчёт по конкурентам | Разведчик (DS) | Оркестратор | Контент-Директор, Стратег, Маркетолог |
| Финальный результат | Оркестратор | — | Тебе |

> DS = выполняет DeepSeek V4 Pro | без пометки = выполняет Claude Opus 4.8

## Типовые сценарии работы

### Сценарий 1 — Еженедельный цикл (каждый понедельник)
```
1. Ты вносишь метрики прошлой недели
2. Аналитик (DS) → отчёт за неделю
3. Разведчик (DS) → свежий мониторинг конкурентов
4. Контент-Директор (DS) → план на неделю
5. Оркестратор → утверждает план, передаёт тебе
```

### Сценарий 2 — Создание единицы контента
```
1. Ты: "Нужен пост на тему X для Instagram, цель — охват"
2. Оркестратор → проверяет по карте смыслов
3. Продюсер (DS) → черновик поста
4. Дизайнер (Claude) → упаковка под Instagram
5. Оркестратор → финальная проверка
6. Тебе: готовый пост
```

### Сценарий 3 — Создание видео-контента
```
1. Ты: "Нужен сценарий для TikTok на тему Y"
2. Разведчик (DS) → что заходит у конкурентов по теме Y
3. Продюсер (DS) → сценарий с хуком и структурой
4. Дизайнер (Claude) → адаптация под TikTok формат
5. Оркестратор → проверка и финал
```

### Сценарий 4 — Адаптация одного материала на все платформы
```
1. Ты даёшь основной материал (видео/статья/мысль)
2. Продюсер (DS) → 6 адаптаций под каждую платформу
3. Дизайнер (Claude) → форматирование под каждую платформу
4. Оркестратор → проверка пакета целиком
5. Тебе: готовый пакет контента
```

### Как работает Регистратор (Claude)

Регистратор — это роль Оркестратора. Он:
1. Принимает задачу от тебя
2. Формулирует чёткое задание для DeepSeek (черновик/рутина)
3. Получает результат от DeepSeek
4. Проверяет на соответствие смыслам и брендбуку
5. Дорабатывает или отправляет на доработку
6. Логирует результат в систему

```
ТЫ → Claude (задача) → DeepSeek (черновик) → Claude (проверка + финал) → ТЫ
```

### Технически: как вызвать DeepSeek из системы

**Вариант A — через OpenRouter API** (рекомендован)
DeepSeek доступен через OpenRouter, Claude Code умеет работать с внешними API через Bash.

**Вариант B — прямой API DeepSeek**
```bash
curl https://api.deepseek.com/chat/completions \
  -H "Authorization: Bearer $DEEPSEEK_API_KEY" \
  -d '{"model": "deepseek-chat", "messages": [...]}'
```

**Вариант C — параллельные сессии**
Claude Code в одном окне, DeepSeek в другом (например, через веб-интерфейс).
Ты передаёшь задание вручную — медленнее, но не требует настройки.

### 8. Брендбук-агент

**Роль:** Создаёт и хранит визуальную и вербальную идентичность. Всё что выходит из системы — соответствует брендбуку.

**Задачи:**
- Сформировать брендбук на основе позиционирования (от Стратега)
- Определить цветовую палитру, шрифты, сетку
- Прописать tone of voice, запрещённые слова, стиль общения
- Создать шаблоны для каждой платформы
- Проверять весь контент на соответствие брендбуку

**Структура брендбука:**
```
1. МИССИЯ И ЦЕННОСТИ
2. АУДИТОРИЯ (портрет, боли, желания)
3. ПОЗИЦИОНИРОВАНИЕ (уникальность, территория)
4. ВИЗУАЛЬНЫЙ СТИЛЬ
   ├─ Логотип и варианты использования
   ├─ Цветовая палитра (HEX / RGB)
   ├─ Типографика (заголовки, текст, акценты)
   └─ Сетка и отступы
5. ВЕРБАЛЬНЫЙ СТИЛЬ
   ├─ Tone of voice
   ├─ Ключевые фразы и формулировки
   └─ Запрещённые слова и клише
6. ШАБЛОНЫ
   ├─ Посты для каждой платформы
   ├─ Обложки и превью
   └─ Документы и презентации
```

- `frontend-design` — визуальные принципы

## Настройка Figma MCP

Figma MCP даёт Дизайнеру/Верстальщику прямой доступ к Figma-файлам:
читать брендбук, извлекать токены, получать компоненты и экспортировать макеты.

### Шаг 1 — Получить Figma Access Token

1. Зайди на [figma.com](https://figma.com) → **Account Settings → Security**
2. Нажми **"Generate new token"** → дай имя "Claude Code MCP"
3. Скопируй токен (показывается один раз)

### Шаг 2 — Добавить токен в окружение

```bash
# Добавить в ~/.zshrc
echo 'export FIGMA_ACCESS_TOKEN="figd_xxxxxxxxxxxx"' >> ~/.zshrc
source ~/.zshrc
```

### Шаг 3 — Подключить MCP-сервер в Claude Code

Добавить в `~/.claude/settings.json` в раздел `mcpServers`:

```json
{
  "mcpServers": {
    "figma": {
      "command": "npx",
      "args": ["-y", "@figma/mcp-server"],
      "env": {
        "FIGMA_ACCESS_TOKEN": "${FIGMA_ACCESS_TOKEN}"
      }
    }
  }
}
```

### Шаг 4 — Получить File ID брендбука в Figma

File ID — это часть URL твоего Figma-файла:
```
https://figma.com/file/[FILE_ID]/Brandbook
                        ^^^^^^^^
```
Сохрани его — передаёшь Дизайнеру при каждом запуске.

### Шаг 5 — Проверить подключение

```
/mcp  ← в Claude Code покажет список подключённых MCP-серверов
```

> **Статус:** 🔲 Требует настройки токена Figma и добавления в settings.json
